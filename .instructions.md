# PR Status Monitor - Agent Development Guide

## What This Extension Does

Monitors GitHub PRs in VS Code, displays status in the status bar, and notifies on changes.

## Extension Activation

- Activation event: `onStartupFinished`
- Entry point: `src/extension.ts::activate()`
- Shutdown: `deactivate()` disposes of timers and resources

## State Management

- `previousPRStatuses: Map<string, string>` - Tracks status changes to trigger notifications
- `isConnected` boolean - Connection state to GitHub API
- `allPRs: PR[]` - Current list of user's open PRs

## Polling Architecture

- Fast polling: 10s (startup/reconnection)
- Normal polling: Configurable (default 2 min)
- Switches between modes based on connection state
- Uses `setInterval()` with clean up on deactivation

## GitHub API Usage

- Uses `@octokit/rest` with VS Code's built-in GitHub auth
- Fetches: check runs, commit statuses, user info
- No personal access tokens in code (uses VS Code's secure auth)

## Commands

- `pr-status-monitor.openPrInBrowser` - Opens PR selection menu or directly opens single PR

## UI Components

- Status bar item (left, priority 100)
- QuickPick menu for PR selection
- Notifications (info/warning)
- Output channel for debugging

## Testing

- Unit tests in `src/test/`
- Run with: `pnpm test`
- Test files: `utils.test.ts`, `extension.test.ts`, `notifications.test.ts`

## Key Functions to Understand

- `activate()` - Extension entry point
- `updatePRStatus()` - Main polling function
- `fetchAndDisplayPRs()` - Fetches PRs and updates UI
- `notifyStatusChange()` - Handles notifications on status transitions
- `determineStatusFromChecks()` - Aggregates check run status
- `determineStatusFromCommitStatuses()` - Aggregates commit status
